extract(pdf_path: str, category: int | None = None, cycle: int | None = None, method: int | None = None)#

从PDF中提取表格数据并写入数据库的入口函数。

参数:

  • pdf_path – pdf文件的绝对路径

  • category – 交易品种，若指定，则以此为准；否则基于PDF文件名自行判断

  • cycle – 交易周期，若指定，则以此为准；否则基于PDF文件名自行判断

  • method – 交易方式，若指定，则以此为准；否则基于PDF文件名自行判断

Note:

  1. 程序自动判断得到的交易品种、交易周期与交易方式的代码可能不准确；

  2. 交易方式实际上要根据表格中的每一行记录来判断，因此，此处指定的 method 参数将会对所有记录生效，除非PDF中只有一行记录，否则可能生成错误的 数据库记录。

返回:

None

summary_sheet_parse.bill_analysis.get_dates(name: str, cycle: int)#

根据账单的名称及交易周期，获取该账单对应的交易日期。

交易日期由两部分构成：

  1. 交易年份

  2. 交易的起止月份（如果为单月交易或旬交易，则起止月份相同）

参数:

  • name – 账单名称

  • cycle – 交易周期的代码

返回:

None 或元组，包含交易年份和交易的起止月份

summary_sheet_parse.bill_analysis.is_legal_pdf(pdf_name: str) → bool#

当前PDF文件是否合法（可被解析）。

根据需求定义，以下交易对应的PDF文件不予解析：

  • 合同转让交易

  • 外送交易

参数:

pdf_name – PDF文件名

返回:

True 或 False
