类型

运行状态标识，可能的值包括：success，和 fail。其中，success 表示分配成功；fail 表示分配失败。

class PowerDistribution#

执行功率分配的类。

_distribute(power: float, units_obj: Dict[str, List[float]], times: int = 0) → None#

实际上执行功率分配的递归函数。

参数:

  • power – 待分配的功率，单位是kW，正值表示放电，负值表示充电

  • units_obj – 用于承接 power 的候选储能单元

返回:

None

distribute(json_data: dict)#

根据传入的数据进行功率分配的主函数。

参数:

json_data – 入参，参数格式

返回:

返回各储能单元的功率分配情况，格式为：{pcs编号: 分配功率} 参考 接口返回值的“data”部分.

every_pcs_is_full() → bool#

经过功率分配后，检验是否每个pcs均达到了最大功率或其SOC达到限制。

如果是，则返回``True``；否则返回那些没有达到最大功率或SOC限制的pcs的编号。

返回:

True: 如果所有pcs均已达到最大功率或SOC限制 UnitsObj: 如果有pcs没有达到最大功率或SOC限制，则返回这些储能单元对象

get_power_lack() → float#

计算各储能单元对象的总功率是否满足待分配功率的要求。

备注

在本应用中，假设待分配的功率及各储能单元分配到的功率均为整数。 由于四舍五入的原因，可能会出现分配后的总功率与待分配功率不一致的情况， 按照调度要求，允许多充/多放，但不允许少充/少放。

待处理

对于浮点数的情况，需要进一步处理。

返回:

如果功率缺额，返回缺额的功率值；否则返回0

update_final_units_obj(units_obj: Dict[str, List[float]]) → None#

更新最终的储能单元对象（self.final_units_obj）的状态值。

参数:

units_obj – 待更新的储能单元对象

返回:

None

para_verify(json_data: dict) → bool#

对请求参数的合法性进行校验。

参数:

json_data – 请求的参数，格式

抛出:

ValueError – 如果参数不合法，则抛出异常

返回:
