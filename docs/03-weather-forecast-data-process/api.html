API#

1、和风API#

本package包含了从 和风气象 获取天气预报（实测）数据并存入数据库的代码.

包含的模块及其主要功能如下：

  • hefeng/168h_forecast_to_platform.py：获取168小时天气预报数据并存入数据库

  • hefeng/30d_forecast_to_platform.py：获取30天天气预报数据并存入数据库

  • hefeng/rt_weather_data_to_platform.py：获取实时天气数据并存入数据库

  • hefeng/resave_csv.py：重新存储csv文件中的数据到数据库

备注

返回:

str, 日志记录器的名称

hefeng.initialize(prod_run: bool, local_run: bool, test_run: bool)#

根据命令行参数, 初始化数据库连接.

参数:

  • prod_run -- 是否在生产环境执行

  • local_run -- 是否在本地执行

2、天机API#

本程序包包含了处理天机气象数据的代码.

包含的模块及其主要功能如下:

  • tianji/csv_2_taos_15d.py: 将15天天气预报数据存入数据库

  • tianji/csv_2_taos_45d.py: 将45天天气预报数据存入数据库

  • tianji/history_2_taos.py: 将历史天气数据存入数据库

  • tianji/init_sqls.py: 初始化数据库表

  • tianji/latest_2_county.py: 将最新的天气数据存入数据库

  • tianji/ori_2_latest.py: 将原始天气数据存入数据库

  • tianji/ori_2_ms.py: 将实时天气数据存入数据库

  • tianji/resume_15d_15min.py: 将15天15分钟天气预报数据存入数据库

tianji.calc_millis_suffix(dir_name: str)#

计算时间戳的后缀.

由于预报数据是滚动更新的, 因此需要通过为相同时刻的预报数据添加不同的时间戳后缀 来区分起报时间.

该时间戳后缀由秒和毫秒构成, 对于类似‘2023112400’的起报日期字符串, 选取其5、6 位数字作为秒, 7、8位数字作为毫秒, 并在毫秒后添加一个0或1, 用于区分是当天的第几次起报.

参数:

dir_name -- str, 类似于‘2023112400’的字符串

返回:

str, 时间戳后缀

3、GFS API#

本package用于处理从美国国家环境预报中心（National Center for Environment Prediction，NCEP） 的全球预报系统（Global Forecast System，GFS）得到的 全球天气数据。

处理的过程分为三步：

  • 从GFS服务器下载数据文件；

  • 将下载的数据文件进行解析入库；

  • 定时保留最新版本的预报数据。

3.1 module 函数#

gfs.get_data#

本module用于从GFS服务器获取天气预报的原始数据。

服务器地址由两部分组成：baseUrl 和 suffix。 本项目只采用空间分辨率为0.25°×0.25°的数据，此项约束在 baseUrl 中进行了设置； 原始数据文件中的数据项和空间分类很多，本项目只采用了其中一部分，此项约束在 suffix 中进行了设置。

具体如下：

数据项：

  • GUST：阵风风速

  • PWAT：降水量

  • RH：相对湿度

  • SUNSD：太阳辐射

  • TCDC：总云量

  • TMP：温度

  • UGRD：风速U分量

  • VGRD：风速V分量

空间分类：

  • 100_m_above_ground：100米高度

  • surface：地表

  • entire_atmosphere：整个大气层

  • entire_atmosphere_(considered_as_a_single_layer)：整个大气层（视为单层）

  • toplat：最大纬度，本项目为54

  • bottomlat：最小纬度，本项目为18

  • leftlon：经度最西侧值，本项目为73

  • rightlon：经度最东侧值，本项目为135

上述经纬度范围覆盖了全国。

GFS每天更新四次预报数据，本项目只选用其中的第一次。

关于全部数据项及其解析的内容，可以参考 此处。

gfs.get_data.download_data(base_url, date_str, time_str, output_dir)#

下载指定日期和时间的GFS原生数据文件。

参数:

  • base_url -- 数据地址

  • date_str -- 日期

  • time_str -- 时刻，本项目中默认为“00”

  • output_dir -- 数据保存目录

返回:

无返回值，通过调用 download_with_retries() 函数将下载的内容写入文件。

gfs.get_data.download_with_retries(url, filepath, retries=3, delay=5)#

带有重试机制的文件下载。

参数:

  • url -- 文件链接

  • filepath -- 下载文件路径

  • retries -- 最大重试次数

  • delay -- 重试间隔，单位：秒

返回:

是否下载成功，True 表示下载成功，False 表示下载失败。

gfs.get_data.get_file_no()#

获取原始文件的编号。

文件一共包含未来16天的预测数据。其中，前5天每小时一个数据点；后11天每3小时一个数据点。 一共有 5*24+11*8=208 个文件。

返回:

文件编号组成的数组。

gfs.get_data.main(date_str, time_str: str = '00')#

下载指定日期、指定时刻的GFS原生数据文件。

参数:

  • date_str -- 预报数据的日期

  • time_str -- 预报数据的时刻，默认从0点开始，即只下载每天第一次预报的数据

返回:

无返回值，预报数据将被写入文件中。
