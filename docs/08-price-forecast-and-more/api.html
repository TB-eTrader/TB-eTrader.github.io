1. 山西#

1.1 现货电价预测#

现货电价按照交易类型可以分为 日前电价 和 实时电价 两类，按照适用结算主体又可以分为 节点电价 （用于发电侧结算）和 统一出清价格 （用于售电侧结算）两类。 上述组合一共有四种，即：日前节点电价、日前统一出清价格、实时节点价格 和 实时统一出清价格。

本算法对上述四种电价进行预测。

对于周期性的定时调度任务而言，每天21点至22点之间程序会预测日前价格；每天11点至12点之间，程序会预测实时价格。 如果指定了 -C 参数，则预测的是统一出清价格；否则预测节点价格。在预测节点价格时， 目前只预测特新场站和晖源场站所在的两个节点。

一些使用示例如下：

# 打印帮助文档
python -m shanxi.price_forecast -h
# 强制预测基准日期为2024-6-1的日前统一出清价格
python -m shanxi.price_forecast -C -F --market 3 -D 2024-6-1
# 预测节点电价，基准日期和市场信息默认进行推断
python -m shanxi.price_forecast -R

备注

数据中可能存在空值，导致无法被利用。为了避免这种情况，读取数据的过程采用了逐日读取的方式。

对于用于训练的数据而言，要求单日的所有基础数据都是完整的，如果存在缺失，则弃用该日的所有数据， 并在一定范围内向前顺移，直至获取到足够天数的数据。

对于用于预测的数据而言，要求单日的新能源出力数据是完整的（即在整个预测日期范围内都有数据），省调负荷和联络线可以缺失， 算法会对缺失数据进行处理。

1.3 旬分时电价预测#

针对山西电力市场的旬分时电价预测模型。

核心仍然是通过供需数据来预测电价数据。供需数据包括：新能源出力、省调负荷以及外送负荷。 由于预测数据的日期较长，交易中心并未全部发布，因此首先需要对这些供需数据进行预测。 算法的具体执行步骤如下：

  1. 通过天气数据来预测长周期的新能源出力数据；

  2. 通过时间序列的方法来预测长周期的省调负荷数据；

  3. 通过时间序列的方法来预测长周期的外送负荷数据；

  4. 基于上述预测的新能源出力，省调负荷以及外送负荷数据来预测长周期电价。

static infer_date(start_date: date, end_date: date) → list[date][源代码]#

确定训练集和预测集的起止日期。

在实际的应用中，一般需要在距离目标旬还有数天的时候预测价格，因此，需要对训练集和预测集的起止日期进行调整。 调整方式为：首先确定训练集的结束日期，然后据此日期来确定其他日期。

确定训练集的结束日期的方式为：获取 start_date 之前的所有的供需数据和价格数据的最新的更新日期的最小值， 并将其作为训练集的结束日期。

参数:

  • start_date – 旬的开始日期

  • end_date – 旬的结束日期

返回:

训练集和预测集的起止日期。
